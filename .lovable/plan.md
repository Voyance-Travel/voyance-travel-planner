
# Rebuild PDF Generator to Match Editorial Design

## Problem
The current PDF generator renders a single giant HTML element, converts it to one big canvas via `html2canvas`, then slices that image across pages. This produces poor text quality (rasterized, not selectable), inconsistent page breaks (content gets cut mid-activity), and a generic "web screenshot dumped into a PDF" look with harsh purple/blue accents and flat styling.

## Reference Design (from the uploaded PDF)
The reference PDF has a premium, editorial travel document feel:
- **Cover page**: Warm cream/linen background (#FAF8F3), "V O Y A N C E" wordmark at top with gold horizontal rules and a diamond ornament at bottom, large serif destination name, gold accent line below, elegant accommodation card with gold "ACCOMMODATION" label
- **Content pages**: "VOYANCE" header with page number on every page, gold horizontal rule beneath header, charcoal day headers (#3D3424) with rounded corners and date right-aligned, gold timestamps (e.g. "15:30"), bold serif activity titles, body description text, muted meta line (location, duration, cost), tip boxes with gold left-border accent and italic text, thin separator lines between activities
- **Locked days**: Charcoal header preserved, dashed border placeholder with lock icon and "Unlock this day at travelwithvoyance.com"
- **Notes page**: Same header, "Notes" title, horizontal ruled lines for writing
- **Footer**: Gold horizontal rule, "Generated by Voyance . travelwithvoyance.com" centered

## Approach: Native jsPDF Drawing (No html2canvas)
Replace the html2canvas approach entirely with direct jsPDF drawing commands. This produces real selectable text, precise page-break control, and a much smaller file size.

### Color Palette
- Background: cream #FAF8F3 (page fill)
- Gold accent: #B8976A
- Charcoal text: #3D3424
- Body text: #555
- Muted text: #999
- Day header background: #3D3424 (white text)
- Locked day header: #9CA3AF

### Typography (jsPDF built-in fonts)
- Destination title: Helvetica Bold, ~28pt
- Section title ("Your Itinerary"): Helvetica Bold, ~20pt
- Day header: Helvetica Bold, ~14pt
- Activity title: Helvetica Bold, ~11pt
- Body/description: Helvetica, ~9pt
- Time stamps: Helvetica Bold, ~10pt, gold color
- Tips: Helvetica Italic, ~8.5pt
- Meta line: Helvetica, ~8pt, muted

## Implementation Details

### File: `src/utils/consumerPdfGenerator.ts` (complete rewrite)

The function signature and `ConsumerTripPdfData` interface stay the same. Internally:

1. **Page setup helper**: Function to draw the cream background rectangle, the "VOYANCE" header with page number, and gold header rule on every page. Also draws the footer rule and "Generated by Voyance" text.

2. **Cursor tracking**: A `y` cursor variable tracks vertical position. Before drawing any element, check if it fits on the remaining page. If not, call `pdf.addPage()`, redraw the page template, and reset `y`.

3. **Cover page**: 
   - Full cream background fill
   - "V O Y A N C E" wordmark centered at top with gold rule lines on either side
   - Large destination name centered
   - Short gold accent line below
   - Date range in muted text
   - Traveler count and day count
   - Accommodation card: rounded rect with light fill, gold "ACCOMMODATION" label, hotel name bold, check-in/out dates
   - Flight card if present: same style, gold "FLIGHT" label
   - Footer with diamond ornament and gold rules
   - "Generated by Voyance" at bottom

4. **Itinerary pages**:
   - "Your Itinerary" section heading with gold underline
   - For each day:
     - **Day header**: Charcoal rounded rectangle spanning the width, "Day N" + theme in white bold, date right-aligned in white
     - **Activities**: For each activity:
       - Time in gold bold (left column, ~15mm wide)
       - Title in charcoal bold
       - Description in body color, word-wrapped
       - Meta line (location, duration, cost) in muted small text
       - Tip box: Gold left-border (2pt line), italic text indented
       - Thin separator line between activities
     - **Page break logic**: Before each activity, estimate its height (title + description lines + meta + tip). If it won't fit, break to a new page.

5. **Locked days**:
   - Charcoal header (same style but gray #9CA3AF)
   - Dashed border rectangle with centered lock icon (black square Unicode) + "Day N is locked" text
   - "Unlock this day at travelwithvoyance.com" in muted text

6. **Notes page**:
   - "Notes" heading
   - ~20 horizontal ruled lines with generous spacing

7. **Page numbering**: Added during page template drawing (top-right corner).

### Text wrapping utility
A helper function that takes text, max width, and font settings, and returns an array of lines using `pdf.splitTextToSize()`. This is critical for descriptions and tips that can span multiple lines.

### Height estimation utility
Before rendering each activity block, calculate approximate height:
- Title: ~6mm
- Description: (number of wrapped lines) x 4mm
- Meta line: ~4mm if present
- Tip box: (number of wrapped lines) x 3.5mm + padding
- Separator: ~4mm
This allows accurate page-break decisions.
