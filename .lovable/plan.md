

## Fix: No Hotel Activities Before Check-in on Day 1

### Problem
On Day 1, the repair pipeline injects hotel activities (Freshen Up, Return to Hotel, mid-day hotel return) **before** the traveler has checked in. The traveler hasn't arrived at the hotel yet, so these make no sense.

Three gaps in the bookend logic:
1. **Step 0 (Morning phantom strip)** — only runs on Day 2+, never strips Day 1 pre-check-in hotel phantoms generated by the AI
2. **Step 1 (Freshen-up injection)** — no Day 1 pre-check-in guard; injects "Freshen up at Hotel" if any hotel-bound transport exists before check-in
3. **Step 1b (Mid-day hotel return)** — already has a `skipBecausePreCheckIn` guard but only for the lunch→dinner gap; doesn't cover other scenarios

### Plan

**File: `supabase/functions/generate-itinerary/pipeline/repair-day.ts`**

#### 1. Add Day 1 pre-check-in guard to Step 1 (freshen-up injection, ~line 2591)
Before injecting a freshen-up card, check if it's Day 1 and the current index `i` is before the check-in activity. If so, skip injection.

```
if (isFirstDay) {
  const checkInIdx = activities.findIndex(a => {
    const t = (a.title || '').toLowerCase();
    return isAccom(a) && (t.includes('check-in') || t.includes('check in') || t.includes('checkin'));
  });
  if (checkInIdx >= 0 && i < checkInIdx) continue;
}
```

#### 2. Extend Step 0 (morning phantom strip) to also cover Day 1
On Day 1, strip any accommodation cards that appear **before** the check-in activity (excluding the check-in itself). This catches AI-generated "Return to Hotel" or "Freshen Up" cards that precede check-in.

Change the guard from `!isFirstDay && !isDepartureDay` to `!isDepartureDay`, then inside the loop, on Day 1 specifically check if the phantom appears before the check-in index.

#### 3. Guard mid-day return (Step 1b) more broadly on Day 1
The existing `skipBecausePreCheckIn` only checks if lunch is before check-in. Extend it: on Day 1, if no check-in activity exists at all, skip mid-day hotel return entirely.
